# 首页模块下一步行动清单（工程决策文档）

## 0. 背景与边界

- 当前首页由 `front-page.php` 负责“模块编排”，模块位于 `template-parts/front-page/`（见 `front-page.php:16-23`）。
- 现阶段主题未引入 ACF（代码内无 `get_field()` / `acf_add_local_field_group()` 等调用），因此“ACF 形态”属于下一阶段能力建设，而非已落地方案。
- 拆分后的各模块基本为 Elementor 静态结构片段，核心风险从“模板可读性”转移到“内容维护与复用治理”。

## 1. 工程合理性审查（合并/拆分/抽象）

### 1.1 拆分是否过细

- `home-hero.php`、`home-stats.php`、`section-about-intro.php` 属于强首页叙事链路，拆分粒度合理；但当前均为纯静态文本 + 静态链接，拆分带来的维护收益有限，后续需要通过“后台可配置”释放价值。
- `section-category-grid.php` 与 `section-industry-grid.php` 都是“三列卡片网格”信息架构，拆成两个文件合理（便于独立改版与复用），但应抽出“卡片数据模型”一致性（标题/描述/图片/跳转/按钮文案）。

### 1.2 应当合并的模块

- 不建议把 `section-category-grid.php` 与 `section-industry-grid.php` 合并为一个文件：二者分别承载“产品分类入口”和“行业应用入口”，未来内容来源（分类 vs 行业/解决方案）与页面复用路径不同，合并会制造耦合。
- `section-feature-media.php` 与 `section-service-grid.php` 也不建议合并：前者偏品牌信任与视频叙事（见 `section-feature-media.php:1-4,169-181`），后者偏能力清单与服务交付（见 `section-service-grid.php:1-16,27-33`）。

### 1.3 目前被标为“首页私有”但应抽成通用的模块

- `section-product-grid.php` 逻辑形态应升级为可复用的 WooCommerce 产品栅格（当前硬编码为一组商品卡片，见 `section-product-grid.php:33-121`），后续可在首页、产品分类页、解决方案页复用同一“产品推荐区块”。
- `section-category-grid.php`/`section-industry-grid.php` 在信息结构上可以复用到 solution / industry landing page，建议抽象为“链接卡片网格”通用模块（数据来自 ACF Repeater 或分类/自定义文章类型查询）。

### 1.4 当前拆分之外的工程风险（必须纳入下一步）

- 首页内存在大量 `index-*.html` 的静态链接（例如 `home-hero.php:55-81`、`home-stats.php:33-38`、`section-feature-media.php:140-145`、`section-project-grid.php:59-66`），会导致本地/线上路径、固定链接、语言站点切换时不可控，属于高优先级技术债。
- `front-page.php` 未编排 `section-industry-grid.php`（文件存在但未引用，见 `front-page.php:16-23` 与 `template-parts/front-page/section-industry-grid.php`），需要明确是否为“计划模块”或“遗留冗余”。

## 2. 模块升级建议（私有 / 复用 / ACF）

决策口径（用于解释“为什么”）：

- 私有：仅服务首页叙事且短期不打算在其他页面出现；改版频率低，或内容稳定到可以写死。
- 复用：信息结构可复用（卡片网格/列表/CTA），且未来会出现在多个模板（blog/solution/product）。
- ACF：内容与跳转需要运营迭代、版本化（多语言/多站点），或需要后台调整顺序/开关/数量；适合用 Group / Repeater 组织。

### 2.1 模块逐个建议

| 模块 | 现状摘要（定位/主要问题） | 推荐形态 | 理由（工程视角） | 预计复用去向 |

## 3. WordPress 实现形态建议（ACF Group/Repeater、写死、未来复用）

### 3.1 适合 ACF Repeater / Group 的模块

- `home-hero.php`：Group（视频 URL、标题、副标题、按钮组、移动端开关）。
- `home-stats.php`：Group + Repeater（指标卡：数字、单位、标题、描述、图标）。
- `section-about-intro.php`：Group + Repeater（卖点列表）。
- `section-category-grid.php` / `section-industry-grid.php`：Repeater（卡片：图片、标题、描述、链接、按钮文案、可选 SVG/Icon）。
- `section-feature-media.php`：Group（视频 URL）+ Repeater（要点/标签）。
- `section-service-grid.php`：Repeater（服务项：标题、描述、图标）。
- `section-project-grid.php`：Repeater（案例：国家、标题、图片、链接）或后续 Projects CPT 查询。

### 3.2 更适合直接写死在模板中的部分

- 仅与主题样式强绑定、且长期稳定不变的结构骨架（Elementor 外层容器与 `data-id` 锚点）应继续写死，避免后台配置导致 DOM 结构漂移，破坏现有 CSS 选择器依赖（如各模块最外层 `section` 的 `elementor-element-*`）。

### 3.3 未来最可能被 blog / solution / product 复用的模块

- `section-product-grid.php`：最强复用候选（推荐产品/相关产品/你可能喜欢）。
- `section-feature-media.php`：Why-us 叙事模块可用于 solution 详情页提升转化。
- `section-service-grid.php`：可作为“交付保障/服务能力”通用区块。
- `section-category-grid.php` / `section-industry-grid.php`：可作为“入口型导航网格”用于汇总页与目录页。
- `section-project-grid.php`：可沉淀为案例 CPT 的“精选案例”组件，覆盖 blog/solution/product 的侧边推荐位或底部推荐位。

## 4. 下一步行动清单（按优先级）

### 高优先级（阻断性风险/强 ROI）

1. 统一清理首页 `index-*.html` 静态链接为 WordPress 可解析链接（避免上线环境断链）。
2. 将 `section-product-grid.php` 从硬编码改为可复用的 Woo 查询模块（数据一致性 + 复用收益最大）。
3. 明确 `section-industry-grid.php` 是否需要编排进 `front-page.php`，否则删除或迁移为通用模块以免漂移。

### 中优先级（运营迭代需求）

1. 为 `home-hero.php`、`home-stats.php`、`section-feature-media.php` 引入 ACF 配置入口（先满足文案/链接/视频可配）。
2. 将 `section-service-grid.php`、`section-project-grid.php` 以 ACF Repeater 实现“可增删改排序”的内容管理。

### 低优先级（结构优化）

1. 抽象“链接卡片网格”通用模块，统一承载 `section-category-grid.php` 与 `section-industry-grid.php` 的数据层。
2. 将 `section-project-grid.php` 升级为 Projects 自定义文章类型（CPT）与归档模板，实现长期内容生产闭环。
